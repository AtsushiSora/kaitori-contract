# 本番運用チェックリスト

## 毎日の運用

- 管理者はSupabase Authの個別アカウントでログインする。
- 共有端末ではブラウザにパスワードを保存しない。
- 作業終了後は必ずログアウトする。
- 契約を完了する前に、お客様控えPDFと店控えPDFの内容を確認する。
- 確認URLと8桁の開封パスコードは別経路で送る。

## バックアップ

- 営業終了後に契約一覧の「JSON出力」を実行し、暗号化された社内保管先へ保存する。
- 本人確認書類はJSONだけでは復元できないため、Supabase DatabaseとStorageのバックアップ状況を別途確認する。
- 月1回、別端末または検証環境でJSON取込と契約PDF表示を確認する。
- バックアップファイルをメールや個人用クラウドへ送らない。

## 削除と保持

- 契約書、本人確認書類、同意記録の保存期間は、税務・古物営業・個人情報保護の要件を確認して社内規程で決定する。
- 保存期間を過ぎた契約は、権限を持つ管理者が契約一覧から削除する。
- 削除後は契約データ、同意記録、本人確認書類がSupabaseから削除されたことを確認する。
- 誤削除に備え、削除前に対象と契約番号を二名で確認する。

## 障害時

- Supabase障害時は新規契約のクラウド保存を中止し、復旧後に再確認する。
- GitHub Pages障害時は契約確認URLの送信を中止する。
- 漏えい、端末紛失、不審なログインを確認した場合は、管理者パスワードを変更し、Supabaseのセッションと監査ログを確認する。

## 外部設定が必要な残作業

- 独自ドメインを取得し、GitHub PagesへDNS設定する。
- 独自ドメインを使用する場合、Supabase Edge Functionの`ALLOWED_ORIGINS`へ新しいOriginを追加する。
- メール自動送信を行う場合、送信サービス、送信元ドメイン、SPF/DKIM/DMARCを設定してから実装する。
- 契約条項、保存期間、個人情報の利用目的は運用開始前に専門家の確認を受ける。
