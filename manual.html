Test Guide

テスト用 使い方説明書

まずはテスト用の架空データで、紙印刷、タブレット署名、メール電子同意の3パターンを確認してください。 実在するお客様情報は、動作確認が終わるまで入力しないでください。

テスト入口を開く 管理ログインを開く 契約作成を開く

0. テスト用ログイン

  1. テスト入口を開きます。
  2. 「テスト用ログインで開始」を押します。
  3. 確認画面でOKを押すと、登録なしで契約作成画面へ進めます。
  4. テスト用ログインは2時間だけ有効です。

本番公開前に、テスト用ログインボタンとテストログイン処理を削除してください。

1. 初回ログイン

  1. 管理ログインを開きます。
  2. 初回は管理者パスコードを設定します。
  3. 確認欄にも同じパスコードを入力して保存します。
  4. ログイン後、自動で契約作成画面へ移動します。

テスト用でも8文字以上のパスコードを使ってください。共有端末では作業後にログアウトしてください。

2. 契約書を作る

  1. 左側の「新規」を押します。
  2. 契約区分を「買取金額あり」または「買取金額0円 / 無償引取」から選びます。
  3. 完結方法を「紙で印刷」「タブレット署名」「メール電子同意」から選びます。
  4. 売主情報、車両情報、金額・引取情報を入力します。
  5. 生年月日、車検満了日、支払予定日、引取予定日は「選択」ボタンからカレンダーを開けます。
  6. 登録番号は「地域名」「分類番号」「かな」「番号」の4つに分けて入力します。
  7. 本人確認書類の写真が必要な場合は、売主情報の「本人確認書類の写真」から表面・裏面などを添付します。
  8. 右側の契約書プレビューに反映される内容を確認します。
  9. 「下書き保存」を押すと、この端末のブラウザ内に保存されます。

3. 紙で印刷してサイン

  1. 完結方法で「紙で印刷」を選びます。
  2. 入力内容と契約書プレビューを確認します。
  3. 「印刷」を押します。
  4. 紙に印刷し、お客様とオーダーオート側で署名または押印します。
  5. 完了後、画面右上の「完了にする」を押します。

4. タブレット署名

  1. 完結方法で「タブレット署名」を選びます。
  2. 画面下部の署名欄にお客様が指またはペンで署名します。
  3. 署名をやり直す場合は「署名クリア」を押します。
  4. 署名後に「印刷」を押して紙またはPDFにします。
  5. 必要に応じてメール添付でお客様へ送ります。

5. メール電子同意

  1. 完結方法で「メール電子同意」を選びます。
  2. 売主のメールアドレスを入力します。
  3. 「暗号化URL生成」を押します。
  4. 「URLコピー」で確認URLをコピーし、お客様へメール送信します。
  5. LINEで送る場合は「LINE文面コピー」を押して、お客様のLINEへ貼り付けます。
  6. 「パスコードコピー」で開封パスコードをコピーし、SMSや電話など別の方法で伝えます。
  7. お客様はURLを開き、パスコード入力後に内容確認と電子署名を行います。

確認URLとパスコードは同じメール・同じLINEに入れないでください。URLは7日で期限切れになります。

6. 契約一覧とバックアップ

  1. 保存した契約は左側の契約一覧に表示されます。
  2. 氏名、車名、契約番号で検索できます。
  3. 「JSON出力」でバックアップファイルを保存できます。
  4. 別端末へ移す場合は「JSON取込」でバックアップファイルを読み込みます。

本人確認書類の写真もJSONバックアップに含まれます。ブラウザの保存データを消すと契約一覧も消えます。テスト後も必要なデータはJSON出力してください。

7. 本人確認書類の写真

  1. 売主情報の「本人確認書類」で種類を選びます。
  2. 「本人確認書類の写真」から画像を選択します。
  3. 表面・裏面など最大4枚まで添付できます。
  4. 間違えた写真は、添付一覧の「削除」で外せます。

写真はこの端末のブラウザ内に保存されます。本人確認書類は機密性が高いため、共有端末ではテスト後に不要データを削除してください。

確認してほしいポイント

  • 買取金額ありの契約書タイトルと金額が正しく表示される。
  • 買取金額0円の契約書で、重量税・自賠責・リサイクル券・自動車税の還付なし文言が出る。
  • 印刷画面で契約書部分だけが表示される。
  • タブレット署名がプレビューと印刷に反映される。
  • 本人確認書類の写真を添付・削除できる。
  • メール電子同意URLに氏名や住所などの平文が入っていない。
  • 別送パスコードでお客様確認ページを開ける。

セキュリティ上の注意

  • このシステムの管理ログインはブラウザ側の保護です。
  • テスト用ログインは本番前に必ず削除してください。
  • 実運用で強い保護が必要な場合は、サーバー側ログインとデータベース保存が必要です。
  • 本番データを扱う前に、管理者パスコードを必ず強いものに変更してください。
  • 本人確認書類の写真は、必要最小限の端末だけで保存してください。
  • お客様確認URLと開封パスコードは必ず別々に送ってください。
  • 共有端末やタブレットでは、作業後に必ずログアウトしてください。
